# Contributing to ServerWatch

Vielen Dank für dein Interesse an ServerWatch! 🎉

## 🐛 Bug Reports

Wenn du einen Bug findest:

1. Prüfe ob der Bug bereits gemeldet wurde (Issues durchsuchen)
2. Erstelle ein neues Issue mit:
   - Klarer Beschreibung des Problems
   - Steps to Reproduce
   - Erwartetes vs. tatsächliches Verhalten
   - Python Version, OS, etc.
   - Log-Auszüge (falls relevant)

## ✨ Feature Requests

Feature-Ideen sind willkommen! Bitte:

1. Beschreibe das Feature im Detail
2. Erkläre den Use-Case
3. Optional: Schlage eine Implementierung vor

## 🔧 Pull Requests

1. Fork das Repository
2. Erstelle einen Feature Branch (`git checkout -b feature/amazing-feature`)
3. Committe deine Änderungen (`git commit -m 'Add amazing feature'`)
4. Push zum Branch (`git push origin feature/amazing-feature`)
5. Öffne einen Pull Request

### Code-Richtlinien

- Folge PEP 8 (Python Style Guide)
- Füge Docstrings zu neuen Funktionen hinzu
- Teste deine Änderungen lokal
- Aktualisiere die README falls nötig

## 📝 Commit Messages

Verwende klare Commit Messages:

```
Add: Neue Funktion X
Fix: Behebt Bug Y
Update: Verbessert Feature Z
Docs: Aktualisiert Dokumentation
```

## 🙏 Danke!

Jeder Beitrag hilft! Auch kleine Verbesserungen an der Dokumentation sind wertvoll.
